การลบช่องว่างออกจากสตริง Python

ช่องว่างเพิ่มเติมอาจทำให้โปรแกรมของคุณต้องชัดเจนขึ้น สำหรับโปรแกรมเมอร์แล้ว สตริง 'python' และ 'python' นั้นดูเหมือนกัน แต่สำหรับโปรแกรมอินเทอร์พรีเตอร์แล้ว ทั้งสองเป็นสตริงที่แตกต่างกัน Python มองเห็นช่องว่างเพิ่มเติมในสตริง 'python' และถือว่ามีความหมาย เว้นแต่คุณจะขอให้คิดอย่างอื่น สิ่งสำคัญคือต้องใส่ใจกับช่องว่าง เนื่องจากบ่อยครั้งที่คุณต้องการเปรียบเทียบสตริงสองสตริงเพื่อพิจารณาว่าเป็นสตริงเดียวกันหรือไม่ ตัวอย่างเช่น ในกรณีสำคัญกรณีหนึ่งอาจเกี่ยวข้องกับการตรวจสอบชื่อผู้ใช้ของผู้คนเมื่อพวกเขาเข้าสู่ระบบในเว็บไซต์ ช่องว่างเพิ่มเติมอาจสร้างความสับสนในสถานการณ์ที่ง่ายกว่ามากได้เช่นกัน โชคดีที่ Python ทำให้การกำจัดช่องว่างที่ไม่จำเป็นออกจากข้อมูลที่ผู้คนป้อนเป็นเรื่องง่าย Python สามารถค้นหาช่องว่างเพิ่มเติมทางด้านขวาและด้านซ้ายของสตริง เพื่อให้แน่ใจว่าไม่มีช่องว่างที่ปลายด้านขวาของสตริง เราจึงใช้เมธอด “rstrip()”

การแก้ไขค่าตัวแปรและเก็บค่าใหม่กลับเข้าไปในตัวแปรเดิมนั้นมักจะทำกันในการเขียนโปรแกรม ซึ่งค่าของตัวแปรสามารถเปลี่ยนแปลงได้เมื่อโปรแกรมถูกดำเนินการหรือตอบสนองต่ออินพุตของผู้ใช้ นอกจากนี้ คุณยังสามารถลบช่องว่างจากด้านซ้ายของสตริงได้โดยใช้เมธอด lstrip() หรือลบช่องว่างจากทั้งสองด้านได้โดยใช้ strip():

โค้ดด้านบนเริ่มต้นด้วยค่าที่มีช่องว่างที่จุดเริ่มต้นและจุดสิ้นสุด จากนั้นเราจะลบช่องว่างพิเศษจากด้านขวาโดยใช้ rstrip() จากด้านซ้ายโดยใช้ lstrip() และจากทั้งสองด้านโดยใช้ strip() การทดสอบฟังก์ชันการสตริปเหล่านี้จะช่วยให้คุณคุ้นเคยกับการจัดการสตริง ในโลกแห่งความเป็นจริง วิธีการสตริปเหล่านี้ใช้เป็นหลักในการล้างข้อมูลอินพุตของผู้ใช้ก่อนจัดเก็บในโปรแกรม

เคล็ดลับในการหลีกเลี่ยงข้อผิดพลาดทางไวยากรณ์เมื่อทำงานกับสตริง

ข้อผิดพลาดประการหนึ่งที่คุณอาจพบเห็นได้เป็นประจำคือข้อผิดพลาดทางไวยากรณ์ ข้อผิดพลาดทางไวยากรณ์เกิดขึ้นเมื่อ Python ไม่รู้จักส่วนหนึ่งของโปรแกรมของคุณว่าเป็นโค้ด Python ที่ถูกต้อง ตัวอย่างเช่น คุณจะสร้างข้อผิดพลาดหากใช้เครื่องหมายอะพอสทรอฟีภายในเครื่องหมายคำพูดเดี่ยว คุณจะได้รับข้อผิดพลาดทางไวยากรณ์ ซึ่งเกิดขึ้นเนื่องจาก Python ตีความทุกอย่างระหว่างเครื่องหมายคำพูดเดี่ยวตัวแรกและเครื่องหมายอะพอสทรอฟีเป็นสตริง จากนั้นจึงพยายามทำความเข้าใจข้อความที่เหลือเป็นนิพจน์ Python ซึ่งสร้างข้อผิดพลาด นี่คือวิธีการใช้เครื่องหมายคำพูดเดี่ยวและเครื่องหมายคำพูดคู่ให้ถูกต้อง:

เครื่องหมายอะพอสทรอฟีอยู่ภายในชุดเครื่องหมายคำพูดคู่ ดังนั้นอินเทอร์พรีเตอร์ Python จึงไม่มีปัญหาในการอ่านสตริงอย่างถูกต้อง

แต่หากคุณใช้เครื่องหมายคำพูดเดี่ยว Python จะไม่สามารถระบุได้ว่าสตริงควรสิ้นสุดที่ใด และคุณจะได้ผลลัพธ์ดังต่อไปนี้

ผลลัพธ์จะแสดงให้เห็นว่าข้อผิดพลาดเกิดขึ้นทันทีหลังจากเครื่องหมายคำพูดเดี่ยวตัวที่สอง ข้อผิดพลาดทางไวยากรณ์นี้หมายความว่าอินเทอร์พรีเตอร์ไม่รับรู้ส่วนหนึ่งของโค้ดว่าเป็นโค้ด Python ที่ถูกต้อง ข้อผิดพลาดอาจมีต้นตอได้หลายสาเหตุ และฉันจะชี้ให้เห็นข้อผิดพลาดทั่วไปบางประการเมื่อเกิดขึ้น คุณอาจพบข้อผิดพลาดทางไวยากรณ์บ่อยครั้งเมื่อคุณเรียนรู้ที่จะเขียนโค้ด Python อย่างถูกต้อง ข้อผิดพลาดทางไวยากรณ์ยังเป็นข้อผิดพลาดที่มีความเฉพาะเจาะจงน้อยที่สุด ทำให้ยากและน่าหงุดหงิดที่จะจดจำและแก้ไข ดังนั้นคุณจึงต้องใส่ใจรายละเอียดอย่างเพียงพอ ในหลายๆ ครั้ง คุณลักษณะการแก้ไขไวยากรณ์อัตโนมัติของตัวแก้ไขโค้ดของคุณจะช่วยให้คุณค้นหาข้อผิดพลาดทางไวยากรณ์บางส่วนได้อย่างรวดเร็วขณะที่คุณเขียนโปรแกรม เราขอขอบคุณที่คุณสละเวลาอ่านบล็อกนี้ เป้าหมายของเราคือการให้ข้อมูลที่ให้มานั้นทั้งให้ข้อมูลและเป็นประโยชน์ต่อคุณ เราจะคอยแจ้งให้คุณทราบถึงความก้าวหน้าล่าสุดและความเข้าใจเกี่ยวกับการเขียนโปรแกรม Python โปรดคอยติดตามเนื้อหาที่น่าสนใจเพิ่มเติมจากเรา ความคิดเห็นของคุณมีค่าอย่างยิ่ง ดังนั้นอย่าลังเลที่จะติดต่อเราหากคุณมีความคิดเห็นหรือคำแนะนำใดๆ ให้เราร่วมเดินทางสู่การเรียนรู้ไปด้วยกัน!

หากต้องการเรียงลำดับรายการสตริงตามความยาวของสตริง คุณสามารถใช้len()ฟังก์ชันนี้สำหรับkeyฟังก์ชันsort() ได้ ตัวอย่างเช่น รายการจะเรียงลำดับตามความยาวของสตริง โดยสตริงที่สั้นที่สุดจะอยู่ก่อนและสตริงที่ยาวที่สุดจะอยู่ท้ายสุด

5.เรียงลำดับสตริงโดยใช้ฟังก์ชัน

ในทำนองเดียวกัน หากคุณต้องการเรียงลำดับรายการสตริงตามค่าจำนวนเต็มของสตริง คุณสามารถแปลงสตริงเป็นจำนวนเต็มได้โดยใช้int()ฟังก์ชันเป็นkeyวิธีsort()การหรือsorted()ฟังก์ชัน ฟังก์ชันนี้จะเรียงลำดับรายการตามตัวเลข

6. เรียงลำดับสตริงจากน้อยไปมาก

หากต้องการเรียงลำดับรายการสตริงจากมากไปน้อยหรือย้อนกลับ คุณสามารถส่งreverse=Trueอาร์กิวเมนต์ไปยังsort()เมธอดหรือsorted()ฟังก์ชันได้ การเรียงลำดับจากมากไปน้อยนั้นตรงกันข้ามกับการเรียงลำดับจากน้อยไปมาก โดยที่องค์ประกอบต่างๆ จะถูกจัดเรียงจากค่าสูงสุดไปยังค่าต่ำสุด (สำหรับสตริง Z ถึง A)
